# 쿼리 튜닝 계획

## 목표
다음 조회 API를 주요 튜닝 대상으로 삼는다.

1. 채팅방 목록 조회
2. 거래 내역 조회
3. 메시지 목록 조회
4. 보유 종목 조회
5. 랭킹 조회

## 공통 접근 방식
- 먼저 단순 구현
- 대량 더미 데이터 생성
- 실제 병목 확인
- 실행 계획 확인
- 인덱스 / 쿼리 구조 / projection / pagination 개선
- before / after 기록

## 튜닝 체크리스트
- 어떤 where 조건이 있는가
- 어떤 order by가 있는가
- 어떤 index를 탈 수 있는가
- count(*)가 반복되는가
- N+1이 발생하는가
- 엔티티 전체 로딩이 필요한가
- offset pagination이 문제를 만드는가

## 기록 항목
- 튜닝 전 SQL
- 튜닝 후 SQL
- 실행 계획 변화
- rows scanned
- 응답 시간 변화
- 적용한 인덱스
