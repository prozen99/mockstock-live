# 프로젝트 개요

## 1. 프로젝트 목적
MockStock Live는 모의주식 투자와 종목별 실시간 채팅을 결합한 서비스입니다.

단순 기능 구현이 아니라 아래 역량을 보여주기 위해 설계했습니다.

- 거래/잔고 정합성
- 실시간 통신 구조 설계
- 대량 조회 최적화
- 모니터링 및 병목 식별
- 성능 개선의 수치화

## 2. 왜 이 주제인가
주식 서비스 형태는 다음과 같은 기술 포인트를 자연스럽게 담을 수 있습니다.

- 시세 스트림: 서버 → 클라이언트 단방향 이벤트
- 채팅: 클라이언트 ↔ 서버 양방향 통신
- 거래: 트랜잭션 정합성
- 보유 종목 / 거래 내역 / 채팅방 목록: 조회 최적화 대상
- 알림 / unread count: 상태 관리와 성능 이슈

## 3. 핵심 설계 방향
- 시세는 SSE
- 채팅은 WebSocket/STOMP
- 거래/보유 자산은 트랜잭션으로 정합성 보호
- 읽기 전용 조회는 최적화된 쿼리로 분리
- 성능 개선은 before / after로 기록

## 4. 포트폴리오 관점의 차별점
이 프로젝트는 일부러 병목이 발생할 수 있는 구조를 먼저 만들고,
실행계획, 응답 시간, 쿼리 수, 연결 수를 기준으로 문제를 식별하고 개선합니다.
