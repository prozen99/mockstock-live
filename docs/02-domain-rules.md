# 도메인 규칙

## 1. 거래 규칙
- 사용자는 보유 현금 이상으로 매수할 수 없다.
- 사용자는 보유 수량 이상으로 매도할 수 없다.
- 매수/매도 시 holdings와 trade_orders는 같은 트랜잭션 안에서 반영된다.
- 거래 체결 가격은 현재 시세 기준으로 계산한다.

## 2. 시세 규칙
- 외부 API를 사용하지 않는다.
- 서버가 정해진 주기마다 종목 가격을 갱신한다.
- 가격 변동률과 거래량은 내부 규칙에 따라 생성한다.

## 3. 채팅 규칙
- 사용자는 종목 채팅방에 참여한 경우만 메시지를 읽거나 보낼 수 있다.
- 채팅방은 종목당 1개를 기본으로 한다.
- 메시지는 저장 후 브로드캐스트한다.
- 채팅방 목록은 마지막 메시지와 unread count를 표시한다.

## 4. 읽음 처리 규칙
- 읽음 처리는 last_read_message_id 기준으로 관리한다.
- unread count는 메시지 개수를 매번 전부 세기보다, 읽음 기준과 마지막 메시지를 활용하는 방향을 우선 고려한다.

## 5. 알림 규칙
- 시세 급등락, 새 메시지, 시스템 이벤트는 알림으로 발행할 수 있다.
- 알림은 SSE 기반으로 단방향 전달한다.

## 6. 네트워크 분리 규칙
- 시세는 SSE
- 채팅은 WebSocket/STOMP
- 일반 조회는 REST
